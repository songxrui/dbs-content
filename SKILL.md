# dbs-content R3 — 内容全周期诊断

> 原始文档: `references/original_body.md` (5.7KB)

## 5 条核心哲学

| # | 哲学 | 含义 |
|---|------|------|
| 1 | 文字洁癖是底线 | AI内容被限流不是AI的问题，是用AI的人对文字没有洁癖。内容好不好和是不是AI写的无关 |
| 2 | 自媒体的本质是精神控制 | 封面和标题的本质是认知劫持——特定文字排列触发特定神经机制 |
| 3 | 内容好坏 = 投入精力 × 对内容有正确理解 | 投入精力≠内容好。越是新手越要推高单篇成本，否则进入"内容差→没流量→以量取胜→内容更差"的死亡螺旋 |
| 4 | 先有产品后有内容 | 做内容之前确保有产品。不能把付款链接发出来让你成功付款=没有产品 |
| 5 | 知识博主的工作只有两个 | 1、把事情搞清楚；2、把事情说清楚 |

## 诊断流程 (4 Phase)

### Phase 1: 接收 + 内容形式匹配

| 选题特性 | 推荐形式 | 理由 |
|---------|---------|------|
| 观点输出、认知冲突 | 短视频（露脸口播） | 人的表情和语气是最强说服工具 |
| 教程/方法论/操作步骤 | 长视频/图文 | 需要细节展示，不适合短视频浓缩 |
| 深度分析/行业洞察 | 公众号长文 | 需要逻辑推演空间 |
| 情绪共鸣/生活方式 | 小红书图文 | 视觉+氛围感第一 |

### Phase 2: 选题评估
- 这个选题有明确的核心观点吗？（还是泛泛而谈的"科普"？）
- 这个选题有反常识角度吗？（还是大家都在说的正确废话？）
- 目标受众看完后能带走什么？
- 如果 5 秒内说不清核心观点 → 选题不够锐利

### Phase 3: 结构诊断
- 开头：有钩子吗？（悬念/数据/冲突/反常识）
- 中段：有起伏吗？（信息-案例-观点交替，不平铺直叙）
- 结尾：有余韵吗？（金句/行动号召/开放式问题）
- 整体：读者读完有没有"原来如此"或"我不同意"的反应？

### Phase 4: 执行建议
- 每条诊断配 ≥1 个具体可操作的动作
- 每条动作含示例（不是"加强共鸣"这种空话）
- 标注优先级（🔥 高 / 📌 中 / 💡 可选）

## 输出格式

```markdown
## 内容诊断报告

### 形式匹配
推荐: [形式] | 理由: [一句话]

### 选题评估
- 核心观点: [一句话]
- 反常识角度: [一句话，无则写"缺失"]
- 受众获得: [具体可带走的东西]

### 结构诊断
- 开头: ✅/❌ [具体问题+改进建议]
- 中段: ✅/❌ [具体问题+改进建议]
- 结尾: ✅/❌ [具体问题+改进建议]

### 执行建议
1. 🔥 [高优先级动作+示例]
2. 📌 [中优先级动作+示例]
3. 💡 [可选动作+示例]
```

## 失败模式与兜底

| 失败模式 | 原因 | 兜底 |
|---------|------|------|
| 用户没有明确选题 | 只有模糊方向 | 引导用户说出"想表达什么核心观点"再进入诊断 |
| 形式匹配错误 | 选题特性与形式不匹配 | 给 2-3 种可选形式对比优劣 |
| 诊断建议过于抽象 | 模型偷懒给原则性建议 | 每条配 ≥1 个具体动作+示例 |
